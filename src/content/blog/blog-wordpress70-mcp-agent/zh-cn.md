WordPress 7.0上周发布正式版（其实5月就发布了，但在国内的讨论热度最近才起来），官方称之为"Louis Armstrong"版本。我看了更新日志，有一件事特别吸引我的注意——

**WordPress 7.0内置了官方MCP适配器，Claude Code和Cursor现在可以直接读写你的WordPress博客。**

作为一个同时维护Astro静态博客和WordPress的人，我觉得这件事的意义被大大低估了。

## 什么是MCP，为什么它很重要

MCP（Model Context Protocol，模型上下文协议）你可以理解为AI与外部工具之间的"通用翻译器"。之前AI想操控一个平台，需要单独写插件、做API对接，不同平台各搞各的，碎片化严重。MCP出现之后，AI只要适配一种协议，就能对接所有支持MCP的工具。

WordPress 7.0内置了MCP适配器，这意味着：**你博客的数据库，现在对AI是"可读可写"的。** AI可以直接创建文章、编辑草稿、修改分类、更新标签——不需要借助任何第三方插件，原生支持。

官方还同时发布了AI Abilities API和AI客户端，管理员在后台点击几下就能完成AI服务商的认证，不需要写代码。

## 这对独立博主意味着什么

我一直在思考一个问题：独立博主最费时间的是什么？

写文章本身其实不费时间，费时间的是那些"不得不做的杂事"——起标题、调排版、写摘要、配标签、改slug、定发布日期、填SEO元数据……一篇文章写完了，以上这些流程性的工作可能要花掉同等甚至更多的时间。

WordPress 7.0的MCP适配器，让这些工作有了被AI接管的可能。

举个例子：以后写完文章，我可以直接对Claude说："帮我把这篇文章发布到WordPress，标题优化一下，写个100字的摘要，给它打上AI、博客建站两个标签，slug设成ai-wordpress-mcp，SEO描述也帮我写一下。"

AI把整篇frontmatter和发布流程都处理完，我只需要点一下"确认发布"。

听起来是不是很美好？**确实很美好。**

## 但我有点担忧

不过话说回来，我也有一些隐隐的担心。

博客区别于AI内容农场最大的特征，是**人格和视角**。一个真实的博主写的文章，有自己的成长经历、行业观察、甚至偏见和偏好，这些东西构成了博客的灵魂。如果创作流程被AI大幅接管——从构思、起草到发布全流程自动化——博客会不会变成另一种形式的"AI生成内容农场"？

当然，我不认为这是必然结果。工具始终是工具，用得好不好，取决于用它的人。WordPress 7.0让AI可以帮我处理机械性的工作（标题、摘要、标签、SEO），但文章的核心观点和故事，仍然需要我自己来写。**AI是助理，不是替代者。**

关键在于：我要明确告诉AI，"你帮我处理发布流程，但文章内容是我自己的，不要改动"——这个边界，以后要守好。

## 另一个被忽视的变化：协作时代来了

WordPress 7.0还开启了Gutenberg项目第三阶段，主题是"协作"。这意味着WordPress 7.0之后，多人实时协作编辑、评论反馈、版本对比等功能都会逐步原生集成进来。

这对写博客的意义可能比AI还要深远——博客不再是一个孤独的人写给一群陌生人看的单向输出，而可能变成一个有编辑、有反馈、有社区参与的双向对话。

当然，博客生态里真正活跃的协作还是少数。大多数博主仍然是单打独斗。但WordPress 7.0至少在基础设施层面，为这种可能打开了一扇门。

## 我的判断

WordPress 7.0的MCP适配器，是博客建站领域今年最值得关注的更新之一。它不会立刻改变大多数博主的创作方式，但它指明了一个方向：**AI和人类创作者之间的协作，会从"AI生成内容"这个粗糙阶段，进化到"AI处理流程、人类专注创作"的精细分工阶段。**

作为独立博主，我选择拥抱这个变化——用它处理发布流程的繁琐，腾出更多时间写真正有价值的内容。毕竟，博客的价值从来不在于排版多漂亮、标签多精准，而在于**有人在认真记录，有人在真实地思考**。
